Prerequisites

Operating system:

Windows Win7 (32bit or 64bit) or Windows XP.


Compiler:

Microsoft Visual Studio 2010 Professional (or Visual Studio 2003 or higher)

Note: The Express Versions of Visual Studio can't be used as OpenTM2 uses the Microsoft Foundation Classes which are not contained in the Express versions


Windows SDK:

Windows SDK 7 (32bit) or Windows SDK 6 (32bit)


Installer

Nullsoft Installer with the FindProcDll plugin (download site: http://nsis.sourceforge.net/Download)


Ensure that the BIN directories of the compiler, the SDK, and Nullsoft installer are contained in your PATH environment variable


Other Packages

Download and unpack the following packages:

  • Axis2/C (download URL http://ws.apache.org/axis2/c/download.cgi)

  • ICU (download URL http://site.icu-project.org/download/50#TOC-ICU4C-Download)

  • Hunspell (download URL http://downloads.sourceforge.net/hunspell/hunspell-1.3.2.tar.gz)

Building OpenTM2

Note that the new OpenTM2 Scripter GUI is not built automatically by the following 2 scenarios. Since the GUI was created as an Eclipse project, you currently have to set up your Eclipse environment and build the jar-file for the GUI from that environment. Refer to Building the OpenTM2 Scripter GUI for further instructions.

Build from the command line

  • Run SETVAR.BAT

  • Run VSVARS32.BAT

  • Run M.BAT.with MASTER as parameter to do a complete build


Per default M.BAT builds the non-debug version, specify DEB as second parameter to build a debug version of OpenTM2.You are free to run the makefile using any script language or build tool.

The build compiles all source files and copies the created binaries, the sample files and the tables into the OTM directory inside the RELEASE directory and runs the Nullsoft installer to create the install package OpenTM2-v.r.m-Setup.EXE (v.r.m = current version, release and modification as defined in the install script; e.g. 0.0.5).

Build from Visual Studio

  • Run SETVAR.BAT

  • Run VSVARS32.BAT

  • Start Visual Studio Workbench (DEVENV.EXE)

  • Create a new workspace project,Type = MAKEFILE

  • as Build command use "m.bat master"

  • Build the project in the workspace

Building the OpenTM2 Scripter GUI

Prerequisites:

  • you are familiar with the usage of the Eclipse IDE

  • a Java runtime environment (JRE) must be installed, e.g. JavaSE-1.6.


Step 1:
Check-out the sources from SVN into your Eclipse workspace. This creates the directory OpenTM2_TestSuite directly under the trunk directory.

Step 2:
Switch to the Java perspective, select the directory OpenTM2_TestSuite and use the context-menu to create a new Java project. In the following wizard settings:

  • leave the field “Project name” empty (will be filled in automatically)

  • uncheck the checkbox “Use default location” and select the directory OpenTM2_TestSuite in your workspace (e.g. “d:\workspace\opentm2\trunk\OpenTM2_TestSuite”)

  • use the button “Finish” to save


Step 3:
Open the Run configurations dialog from the “Run as” item in the context-menu. Double-click on “Java application”and select the main type “Main – de.ibm.com.opentm2testsuite.gui” (should be selected already).

Step 4:
Export the project as a “Runnable JAR file”:

  • Launch configuration: “Main – OpenTM2_TestSuite”

  • Export destination: navigate to the OpenTM2_TestSuite directory (see above), for the file name use “OpenTM2ScripterGUI.jar”

  • Library handling: select the second choice “Package required libraries into generated JAR” (this will add the JDOM JAR-file to our JAR)

  • use the button “Finish” to save
